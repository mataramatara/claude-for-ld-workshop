Part Eight · Work smarterUsing Claude without burning through your plan.

A few habits that stretch your plan further — and improve your results.

01 · The Go Word

Add this to your Personal Preferences: "Before you build or draft anything, ask me clarifying questions. I'll say GO when I'm ready."

This prevents Claude from generating a full output before you've given it all the context — which means you won't spend a message correcting a response that was already wrong. Fewer iterations. Better results on the first try.

02 · Be specific about format and length

Shorter prompts asking for shorter outputs use fewer resources and often produce cleaner results.

Add a length constraint: "150 words or fewer," "three bullet points," "one paragraph." You'll get what you need without the filler.

03 · Use Markdown files to save on usage

What is a Markdown file? A .md file is plain text with simple formatting symbols — # for headings, ** for bold, - for lists. Claude reads and writes Markdown naturally.

Why it saves usage: When you save context as a .md file in your Project knowledge, Claude can reference it precisely without you re-explaining in each chat. Shorter conversations, better responses, fewer wasted messages.

Best for: Instructions, style guides, research notes, outlines, reference material — anything you'd otherwise paste repeatedly. Create one in any text editor and save as filename.md.

04 · Use NotebookLM for research, Claude for building

NotebookLM (Google, free and unlimited) is exceptional for uploading source documents and synthesizing research.

Do your research and synthesis there. Then bring the polished summary into Claude when you're ready to design or build. Two tools, each doing what it does best.

Part Nine · ConnectorsBefore you authorize a Connector.

When you connect Claude to an external tool, you're giving it access to read — and sometimes write — in that tool.

  1. Only enable what you're actively using. Less access is always safer — re-enable when you're ready to use it.
  2. Check what level of access each Connector requests. Some are read-only. Others are read-write (Claude can also create, edit, or delete). Know the difference before you click Authorize.
  3. For workplace tools — Slack, Google Drive, Gmail — think it through. Are you comfortable with Claude being able to see everything in this account? If not, use a personal account or scope the access.
  4. Turn off Connectors between sessions. You don't need them running in the background when you're not actively working.
  5. If something feels off, revoke the access. You can disconnect any Connector at any time from the Connectors settings. No harm done.
Connectors make Claude dramatically more powerful. The caution here isn't about avoiding them — it's about using them intentionally.

Part Ten · GitHubWhat GitHub is. Why it matters. How hard it is.

You don't need to know how to code to use GitHub. You need to know it exists and what it gives you for free.

What it is

A place to store projects and code — think Google Drive, but for websites and apps. It keeps a full version history of everything you've ever saved.

What it gives you

GitHub Pages — free website hosting. Any site you build with Claude can go live at a public URL with a few clicks. No monthly hosting fees.

How hard it is

I log on, click a button or two, and leave the technical stuff to learn later. You don't need to be technical to start.

The three things to know

  1. GitHub is free to sign up. Free private and public repositories, free version history, free GitHub Pages hosting. Start at github.com.
  2. GitHub Pages makes your Artifact shareable without a Claude account. Upload an HTML file and share the link with anyone — they don't need Claude to view it. This is how you get a real standalone deliverable out the door.
  3. Version history is your undo button. Every time you save, GitHub keeps the old version. If something breaks, you can go back.
If you're curious about GitHub, ask Claude to walk you through setting it up. It will tell you exactly what to do, step by step, and explain the purpose of each step as you go.
Hidden gem: GitHub is full of people sharing their ideas, documentation, workflows, and prompts in public repositories — for free. Search GitHub for topics like "instructional design prompts," "Claude workflows," or "L&D AI tools" and you'll find real practitioners sharing what actually works. You can browse, copy, and adapt anything in a public repo. It's one of the most underused learning resources out there.
