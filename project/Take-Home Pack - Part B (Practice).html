A seven-day challenge to build the habit, ten small moves that compound over time, and a reflection sheet to close the week with intention.

Part Two · 7-day challengeOne small thing, every day.

Print this page. Tape it to your monitor. Check a box every day. Seven reps beats one ambitious Saturday.

DAY 01 · Monday

Draft one email with Claude.

Any email. Paste what you know, let Claude draft, then edit. Notice how long it took vs. writing from scratch.

DAY 02 · Tuesday

Interrogate a long doc.

Paste something dense. Ask: "What are 3 questions this raises for a new manager?" Save the questions.

DAY 03 · Wednesday

Generate 5 MCQs. Critique them.

Pick a topic you know cold. Use Prompt 06. Then use Prompt 07 on Claude's own output. Notice what it catches.

DAY 04 · Thursday

Start one Project.

Create a Project for something you're working on this month. Pin one reference file. Write one sentence of instructions.

DAY 05 · Friday

Make one job aid as an Artifact.

Use Prompt 09. Iterate twice. Send it to a colleague for feedback.

DAY 06 · Saturday

Let Claude critique you.

Paste something you wrote last week. Ask: "What's weakest about this? Be direct." Sit with the answer.

DAY 07 · Sunday

Teach one colleague one thing you learned.

Five minutes at the coffee machine. You'll remember it twice as well, and now there are two of you. That's how this gets into your team.

Part Three · Ten habitsSmall moves that compound.

These aren't prompts — they're the posture. Adopt one a week and your outputs will look different in a month.

01
One task per prompt.

If a prompt is doing three jobs, split it. Focused prompts outperform compound ones — every time.

02
Always add success criteria.

"It's good if X, Y, Z." Claude hits what you name. If you don't name it, you get a guess.

03
Front-load context.

Give everything Claude needs up front. Trickling context costs messages and quality.

04
Stop when it's off track.

Don't keep nudging a drifting conversation. Start a new chat with a cleaner prompt.

05
Ask for the method, not the answer.

"Walk me through how you'd approach this" gets you a better answer than "give me the answer."

06
Tell Claude to push back.

"Disagree with me if I'm wrong. Point out what I'm missing." Its default is agreeable — override that.

07
Test before you share.

Run the prompt yourself before passing it to a teammate. Half the time, you'll edit it first.

08
Don't blindly trust outputs.

Especially financial, regulatory, or citation-level detail. Verify anything a stakeholder will read.

09
Iterate in one-sentence edits.

"Shorter." "More skeptical." "Cut the filler." Small moves beat rewriting the whole prompt.

10
Save what works.

Keep a doc of your ten best prompts. Reuse, don't rewrite. This single habit doubles your output.

Part Four · ReflectionEnd of week one.

After the seven days, sit with this sheet for ten minutes. Write by hand if you can.

Which prompt surprised me — in a good way or a bad one?
Where did Claude save me real time this week? Be specific: which task, how much time.
Where did Claude waste my time or produce something I had to mostly throw out?
What's one recurring L&D task I now want to always run through Claude?
What's one task I decided Claude should never touch? Why?
Who on my team am I going to show this to next week?
